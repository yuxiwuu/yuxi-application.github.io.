Q1 個人簡介和申請動機

我是吳昱希，大學在實踐大學主修工業產品設計，畢業後轉職成為 UI/UX Junior Designer 約一年半，參與過網站 UI 介面設計、功能優化與 UX 研究等專案。這些經驗讓我對「如何創造高互動性的數位產品體驗」產生興趣，也培養了我在設計流程與使用者思維上的能力。

在專案中，我曾使用 Webflow 製作網站，雖然它是一個設計師能快速上手的工具，但在操作過程中，我對「背後的程式邏輯」產生強烈的好奇，也很享受把設計轉化為互動介面的過程。與工程師的合作也讓我意識到，自己不只想停留在設計端，而是希望能獨立完成產品、深入理解技術實作。這段經驗讓我下定決心轉職成為工程師。

目前我已離職並全職投入轉職準備，計劃透過這半年的訓練營建立工程基礎並累積實戰作品。我期待將設計思維與工程能力結合，成為一位能獨立開發、同時理解設計語言的工程師。我相信這樣的跨領域能力，不僅能提升團隊合作效率，也能幫助我在未來的職涯中創造更大價值。

我目前透過 Udemy 全端課程 與 彭彭老師 YouTube 教學自學，學習了 HTML、CSS 與部分 JavaScript 基礎。目前還沒有正式作品，主要都是課程練習的內容為主，希望能在訓練營的過程中累積專案經驗，建立自己的作品集，作為轉職的重要基礎。

Q3 如果參與這個訓練，會怎麼安排學習時間？

我目前是全職學習的狀態，因此可以安排每天 6-8 小時以上專注在學習與實作上。我的規劃是平日每天早上 2 小時、下午 5 小時，專心進行每週的任務。晚上則保留 1–2 小時作為彈性時間，處理當天還沒解決的問題，或是整理筆記與查資料。週末我會花一天時間複習、加強自己薄弱的部分，並且檢查每週任務是否完整，目前還不太確定學習的模式實際操作的狀態，但理想上會想要將一週的時間拆分成2-3階段，分別先完成基本架構，後研究、查找資料解決卡關的地方，最後檢查、調整細節

Q4 軟體技術日新月異，如何確定投入的領域是正確有回報的？

我認為沒有任何一個技術或領域會是絕對正確的選擇。軟體產業的變化非常快，因此與其糾結投入的是不是「正確」，更重要的是建立穩固的基礎，並保持彈性去面對不同的挑戰，以核心基礎支撐未來學習新工具的門檻，並保持開放的心態，不斷接受新資訊，並在技術更替時，持續學習、更新自己的能力。對我而言，「正確」不是指挑到一個穩定有發展性的領域，而是養成持續學習的習慣與能力。只要能堅持進步，就能跟上時代的變化，並在不同的新領域找到發揮的機會

Q5 請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

大二的產品設計課接近期末時，我挑戰設計並製作一台 1:1 的工具推車，連續熬夜三週後，卻在評圖前 10 分鐘組裝時，骨架在鎖螺絲的瞬間裂開，導致整台推車歪斜不穩，當下情緒非常崩潰，甚至差點因此被當掉。

好好睡了一覺後，隔天冷靜下來，我決定重新補救，我趁著最後一週開始重新發想設計，在短時間內完成了一個規模較小但完整的新作品，包括概念發想、建模、3D 列印、表面處理與噴漆，最後成功趕出來重新打分數。我覺得在面對強烈的負面情緒時，先抒發並釋放情緒很重要，等到這些低落的思緒沉澱之後，可以比較冷靜、實際的面對困難和解決問題。

Q6 關於這份申請網頁，分享一個開發時的技術心得。

這次製作頁面的時候，我遇到一個小問題，設定的 :hover 效果一直出不來，一開始以為是語法寫錯，檢查了很久才發現是 :hover 前面多了一個空格。刪掉之後就順利成功了。

這次經驗讓我意識到，程式常常會因為很小的細節而出錯，但只要耐心檢查，最後還是能找到問題並解決。雖然這次只是個小問題，但感覺未來一定還會遇到更多類似的狀況、更多細節需要注意，所以提醒自己在寫程式時要更細心、嚴謹，避免因為小錯誤浪費太多時間。

Q7 如何看待人工智慧的發展對軟體工程師的影響？

我認為人工智慧的發展確實讓軟體工程師的工作型態出現變化。AI 工具可以協助完成一些重複性的程式碼撰寫與除錯，讓效率大幅提升。但工具都有兩面性，錯誤的使用也會相對造成更大的問題，最近 Vibe Coding API 翻車的事件就是個很好的警示，如果沒有理解基礎原理與驗證過程，就盲目依賴 AI 產生的結果，會造成嚴重的安全漏洞與成本風險。

對我來說，AI 是一個有力的輔助工具，最重要的還是要打好基礎，具備獨立判斷力與問題解決能力，才能正確引導 AI 的使用，確保專案能穩定、安全地完成。
